# GrowLog

## 1. サービス概要

GrowLogは、日々の努力の積み重ねを記録し、  
その積み重ねを木の成長として可視化する継続支援アプリです。

努力をしてもすぐに成果が見えないとモチベーションが下がってしまう人が、  
自分の努力を視覚的に実感しながら継続できることを目指したサービスです。

---

## 2. このアイデアはどこから生まれたか

### 2-1. きっかけとなった体験・感情

自分は努力することが苦手で、目標を決めて努力しても途中で辞めてしまうことが多くありました。

特に、努力の結果がすぐに見えないものに対しては達成感を感じにくく、  
モチベーションを保つことが難しいと感じていました。

努力してもすぐに成果が見えないと、「意味があるのだろうか」と感じてしまい、  
結果として継続できずに終わってしまうことが多くありました。

そこで、どんな小さな努力でも達成感を感じられる仕組みを作れないかと考えたことが、このアプリのアイデアのきっかけです。

---

### 2-2. なぜそれが「気になった」のか

自分は「自分を変えたい」と思い、目標を立てることがあります。  
しかし、目標を達成できないことが多く、後から「あの時努力していれば」と後悔することがよくありました。

本当は努力したいと思っているのに、それを続けることができない。  
そんな自分を少しずつでも変えていける仕組みがあればいいと思いました。

そのため、努力の結果ではなく、努力したという事実そのものを実感できるアプリを作りたいと考えました。

---

## 3. 課題の整理（表面的になっていないか）

### 3-1. 表に見えている困りごと

目標を立てて努力を始めても、努力の成果はすぐには見えません。

また、達成感を得られるのは目標を達成した時だけであり、  
長期的な目標では途中で達成感を感じにくく、モチベーションを保つことが難しくなります。

その結果、努力を続けることができず、途中で挫折してしまうことが多くありました。

---

### 3-2. 本当に解決したい課題は何か

- なぜ？ → 自分が成長できない  
- なぜ？ → 努力を続けることができない  
- なぜ？ → 自分を変えたい時に、変わるための努力を続けることができない  

→ 本当に解決したい課題：努力しているのに、その積み重ねを実感できず、継続できなくなってしまうこと。

## 4. 想定ユーザーについて

### 4-1. 想定しているユーザー

**年齢設定**

- 年齢：10代〜30代
- 理由：勉強、資格取得、自己成長など、新しいことに挑戦したり目標を立てて努力する機会が多い年代であるためです。

**生活・状況設定**

- 生活・状況：勉強、資格取得、筋トレ、プログラミング学習など、自分を成長させるために努力をしたいと思っている人
- 理由：努力を続けたいと思っていても、成果がすぐに見えないことでモチベーションが下がり、継続できなくなることがあるためです。

このアプリでは、努力の積み重ねを木の成長として可視化することで、
努力している実感を得ながら継続しやすくなることを目指しています。

### 4-2. 自分とユーザーの距離

- 自分自身

理由：自分自身が努力を継続することが苦手であり、同じ悩みを持っているためです。

---

### 4-3. 実際に使われる可能性について

- まず最初に使ってくれそうな人  
  → 勉強や資格取得、プログラミング学習、筋トレなど、自分を成長させるために努力を継続したいと思っている人

- その人はどんな場面でこのアプリを思い出すか  
→ 何か新しい目標に挑戦しようと思った時  
→ 自分の努力の積み重ねを実感したいと感じた時

- 今の生活の中で代わりに使っているもの  
  → 習慣管理アプリ（Habitify）や日記アプリ

- それを置き換えてまで使う理由  
  → 日記のように文章を書く必要がなく、簡単に努力を記録できる  
  → 木の成長や努力した日数で、努力の積み重ねを実感できる

---

### 4-4. ユーザーがサービスを導入して利用するまでのイメージ(導入・継続の流れ)

このサービスは、目標を立てて「努力して自分を変えたい」と思ったときに、
継続するためのサポートとして利用されることを想定しています。

- どんな場面・困りごとがきっかけで、このサービスを知る・思い出すか  
  目標を立てて努力を始めようとしたときや、努力を続けたいと思っているときに、  
  インターネットで習慣化アプリや努力記録アプリを探す中でこのサービスを知ることを想定しています。  
  また、「努力を続けたいがモチベーションが続かない」と感じたときにも思い出してもらうことを想定しています。

- 最初の一歩（登録・導入）で、ユーザーは何をする必要があるか  
  まずユーザー登録を行い、その後自分の目標を登録します。  
  目標に対して努力を達成した日は「達成」ボタンを押すことで努力が記録され、木が成長していく仕組みになっています。

- 利用を続ける理由は何か？（習慣・必要性・価値）  
  努力を記録するたびに木が少しずつ成長するため、努力の積み重ねを視覚的に実感できます。  
  木の成長を見ることで達成感を得られ、「また成長させたい」という気持ちが継続のモチベーションにつながると考えています。  
  また、毎日の小さな努力でも積み重ねとして可視化されるため、努力を続ける習慣づくりにも役立つと考えています。


## 5. 既存サービス・競合調査

### 5-1. 似たサービス

- サービス名：DotHabit
- URL：https://dothabit.com/
- どんなことができるか：「DotHabit」は、運動や勉強など続けたい習慣を登録し、達成した日をドットで記録・可視化しながら、通知や振り返り機能で継続をサポートする習慣管理アプリ。

- サービス名：Todoist
- URL：https://www.todoist.com/ja
- どんなことができるか：「Todoist」は、仕事や日常のやるべきこと（タスク）を登録・整理し、期限やリマインダー、プロジェクト管理などで効率よくタスク管理できるToDoリストアプリ。

- サービス名：Habitica
- URL：https://habitica.com/static/home
- どんなことができるか：「Habitica」は、やるべきタスクや習慣を登録し、達成すると経験値や報酬がもらえるRPGゲームのような仕組みで、日常の目標達成や習慣化を楽しく続けられるタスク管理アプリ。

---

### 5-2. それでも自分が作りたい理由

既存の習慣化アプリでは、カレンダー形式で記録するものが多いです。

カレンダーが埋まっていく達成感はあるかもしれませんが、  
記録できなかった日があるとカレンダーが崩れてしまいます。

その結果

- 「もう綺麗に埋まらないから来週からやろう」
- 「来月からやろう」

と感じてしまい、努力を続けるモチベーションが下がると感じました。

GrowLogでは、毎日続けられなくても、努力した回数が増えることで木が成長していくため、  
自分の努力の積み重ねを実感できる仕組みを作りたいと考えています。

---

### 5-3. 差別化ポイント

**これらのサービスとの違い**

既存の習慣管理アプリでは、カレンダー形式やタスクリスト形式で記録するものが多く、
「毎日続けること」を前提とした設計になっていることが多いです。

しかしGrowLogでは、毎日続けられなくても努力した回数の積み重ねを可視化することを重視しています。

努力を記録すると木が成長する仕組みにすることで、
結果だけではなく「努力している過程そのもの」を実感できる体験を提供したいと考えています。

## 8. このアプリの懸念点とその対策

### 懸念点

木がある程度成長すると、成長の変化がわかりにくくなる可能性があります。

**理由**

樹齢1年の木と樹齢100年の木では大きさの違いが分かりやすいですが、  
樹齢100年と120年では見た目の変化が分かりにくくなるためです。

**対策**

木の高さを比較できる目盛りをつけたり、  
建物など比較できる背景を追加することで、成長を実感しやすくします。

## 10. 技術スタック

###　10-1. 使用予定の技術

- フレームワーク：Ruby on Rails
- DB：PostgreSQL
- デプロイ先：Render
- 使用予定ライブラリ：
  - Devise
  - OmniAuth（Googleログイン）
  - Hotwire（Turbo / Stimulus）
  - Tailwind CSS

---

### 10-2. 技術選定の理由

#### なぜこの技術を使うのか

GrowLogは、ユーザー登録、目標登録、努力記録、木の成長表示など、
CRUDを中心とした機能を持つWebアプリであるため、
データ管理や画面作成を効率よく行えるRuby on Railsを採用したいと考えています。

また、DBにはPostgreSQLを使用する予定です。
データの整合性を保ちやすく、Railsとの相性もよいためです。

認証機能にはDeviseを利用し、
ユーザー登録やログイン機能を効率よく実装したいと考えています。

さらに、GrowLogでは「努力を記録したら木が育つ」という体験が重要になるため、
Hotwire（Turbo / Stimulus）を用いて、
画面遷移や一部の動きをなめらかにしながら実装したいと考えています。

デプロイ先にはRenderを想定しています。
RailsとPostgreSQLの構成で公開しやすく、個人開発でも扱いやすいと考えたためです。

#### 今回チャレンジしたい点

今回は、単なる記録アプリではなく、
努力の積み重ねを木の成長として可視化する体験づくりに挑戦したいです。

そのために、努力記録に応じて画面上の木が変化する仕組みや、
継続したくなるUI/UXを意識して実装したいと考えています。

また、Hotwireを活用して、
大きなフロントエンドフレームワークを使わずに
気持ちよく操作できる画面を作ることにも挑戦したいです。

#### 不安な点

木の成長をどのような条件で変化させるか、
またその変化をユーザーにとって分かりやすく表現できるかに不安があります。

さらに、機能を増やしすぎるとMVPの開発が重くなるため、
まずは最小限の機能で形にし、
その後に改善を重ねていく必要があると考えています。

通知機能や日記機能など、
本リリースで追加したい要素についても、
実装の優先順位を整理しながら進めたいです。
